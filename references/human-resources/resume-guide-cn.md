# HR 岗位简历写作指南

> 来源：资深 HR 招聘官视角 —— 写简历的底层逻辑

## 核心原则

### HR 看 HR 简历的底层逻辑

当 HR 筛选同行的简历时，内心判断标准是：

| 简历上的说法 | HR 的实际质疑 |
|---|---|
| "会招聘" | 招到过多少人？周期多长？质量如何？ |
| "懂员工关系" | 处理过什么棘手案例？仲裁？辞退？工伤？ |
| "做了薪酬" | 覆盖多少人？什么系统？出错率？ |
| "有 BP 思维" | 你懂业务吗？支持哪个部门？推动业务方做了什么？ |

**结论：** 不要写"负责招聘""负责培训"这种职责描述。要写你**解决了什么问题、产出了什么结果、沉淀了什么方法**。

### 万能公式

```
【动作对象】+【量化结果】+【核心方法/工具】+【可验证的改善】
```

## 各模块写法拆解

### 招聘模块

**差的写法：**
> 负责公司各岗位的招聘工作，包括发布职位、筛选简历、安排面试、跟进 offer，满足各部门用人需求。

**好的写法：**

```
招聘交付 | 年度累计到岗 86 人（含销售/运营/技术），平均周期 18 天，试用期通过率 92%
攻坚岗位：主导 3 个区域 BD 经理招聘，Mapping + 竞品人选池搭建，35 天入职 4 人
渠道优化：停用低 ROI 平台（4500 元/人），转垂直社群 + 内推激励，内推贡献率 15% → 41%，年省 7.2 万
流程提效：建立结构化面试题库，面试官满意度 67% → 89%，新人达标率 +23%
```

**着重写什么：**
- 结果前置：到岗人数 + 周期 + 质量（试用期通过率）
- 攻坚案例：难招的岗位怎么破局（Mapping / 竞品 / 新渠道）
- 数据闭环：成本节省、效率提升、留存改善

### 员工关系 / 劳动关系

**差的写法：**
> 负责员工入转调离手续办理，管理员工档案，处理员工咨询，协助处理劳动纠纷。

**好的写法：**

```
劳动关系风控 | 优化全周期流程，离职争议率 12% → 2.3%
棘手案例：独立主导试用期不胜任解除案，补全证据链，协商解除，0 赔偿
流程再造：发现岗位说明书签字缺失率 40%，推动电子签 + SOP，合规率 → 100%
离职管理：设计离职面谈"风险信号"分级表，预警 8 起纠纷，5 起转协商一致
```

**着重写什么：**
- 具体案件类型："试用期不胜任解除""违纪辞退""工伤谈判"
- 你的角色：独立主导还是协助？用了什么法律条款和证据链？
- 制度输出：SOP、清单、分级表 —— 把个案经验变成公司规则

### HRBP / 综合型岗位

**差的写法：**
> 支持技术部门 HR 工作，包括招聘、绩效、员工关系、团建等，协助业务负责人进行团队管理。

**好的写法：**

```
HRBP | 支持研发中心 120 人（Java / 产品 / 测试）
组织诊断：离职数据分析发现入职 6-12 月后端开发离职率 35%，根因导师制虚设。
  推动技术骨干 1 对 1 带教 + 绩效挂钩，半年内离职率 → 12%
人才盘点：组织技术序列能力校准会，识别 8 名高潜（3 人晋升），2 人 PIP（1 人留任）
业务协同：针对交付延期痛点，设计项目奖与个人绩效解耦方案，交付率 58% → 79%
```

**着重写什么：**
- 业务语言：离职率、交付率、晋升成功率 —— 不是"做了团建""组织了培训"
- 诊断 → 动作 → 结果：发现问题 → 解决方案 → 量化改善
- 对业务 leader 的影响力：你推动业务方做了什么决定

### 培训 / 发展模块

**好的写法示例：**

```
新员工培训 | 覆盖 120 人，培训后 3 个月留存率 68% → 89%
课程体系：3 天纯授课 → "2 天课程 + 1 天业务实战沙盘"，NPS 6.2 → 8.5
导师积分制：带教 1 人积 5 分可兑评优加分，导师参与率 40% → 95%
```

## 高频关键词（ATS）

| 模块 | 关键词 |
|------|--------|
| 招聘 | 招聘交付, 人才寻访, Mapping, 雇主品牌, 结构化面试, 招聘 ROI, 渠道优化, 内推体系 |
| 员工关系 | 劳动关系, 入转调离, 合规管理, 离职面谈, 劳动仲裁, 工伤处理, 风险预警, SOP |
| 薪酬绩效 | 薪酬设计, 宽带薪酬, 绩效考核, OKR, KPI, 奖金方案, 薪酬调查, 人力成本 |
| 培训发展 | 培训体系, 导师制, 人才盘点, 继任计划, 胜任力模型, 学习发展, 企业文化 |
| HRBP | 业务协同, 组织诊断, 人才梯队, 组织效能, 变革管理, 员工敬业度, 团队融合 |
| 数据分析 | Excel, Power BI, 离职预测, 人力报表, 人效分析, 招聘漏斗 |

## 三个不要写 vs 三个必须写

| 不要写（HR 一眼识破） | 必须写（体现专业度） |
|---|---|
| "负责招聘全流程" | "年度招聘交付 XX 人，平均到岗周期 XX 天" |
| "熟练使用办公软件" | "用 Excel 搭建离职预测模型（准确率 82%）" |
| "协助处理员工关系" | "独立主导 XX 劳动争议，为公司节省 XX 元 / 避免赔偿" |

## 不同公司的侧重点

| 公司类型 | 看重什么 | 简历侧重点 |
|---|---|---|
| 大厂 / 正规企业 | 数据能力、合规意识 | 劳动争议案例、数据驱动决策 |
| 中小公司 | 直接上手、搞定难招岗位 | 招聘攻坚、一人多能的系统化表达 |

---

## ATS 策略

简历可能经过机器筛选或 HR 快速浏览：

1. 从对应模块的高频关键词中选 5-10 个与自己匹配的
2. 关键词要**自然融入**经历描述，不要堆砌在末尾
3. 文件名用 `姓名-职位-工作年限.pdf`（不用 `.doc`，排版易错乱）
